1 プロジェクト振り返り

企画・設計段階の振り返り 必須
マーケットリサーチ、ペルソナ設計、情報設計について

企画・設計段階では、市場調査を通して「シーンから花を選べる」というBloom Letterの強みを明確にできたことが良かったです。また、ターゲット設定やサイトマップ、ワイヤーフレームを先に整理したことで、必要なページ構成や導線を具体化できました。一方で、デザイン性と使いやすさを両立しながら、機能の優先度を整理する点には苦労しました。次回は、企画段階で優先順位をさらに明確にし、より効率よく設計へつなげたいです。

デザイン段階の振り返り 必須
デザインガイドライン、ワイヤーフレーム、Figma作業について

デザインガイドライン
デザインガイドラインの作成では、配色やタイポグラフィ、余白のルールを整理することで、サイト全体の統一感を持たせられたことが良かったです。実装時にも判断基準として活用でき、デザインの方向性を保ちながら作業を進められました。一方で、見やすさと世界観の両立には苦労し、細かな調整に時間がかかりました。次回は、運用しやすいルールとして整理し、より効率よく実装につなげたいです。

ワイヤーフレーム
ワイヤーフレーム作成では、各ページに必要な情報や導線を整理し、実装前に全体像を明確にできたことが良かったです。情報の優先順位を意識しながら構成を考えたことで、ユーザーが迷いにくい画面設計につながりました。一方で、限られた画面内に必要な情報を収める調整には苦労しました。次回は、レスポンシブも含めた構成を早めに検討し、より実装しやすい設計にしたいです。

Figma作業
Figma作業では、完成イメージを具体的に可視化しながら、レイアウトや配色を事前に調整できたことが良かったです。実装前に画面の方向性を固められたため、作業中の迷いを減らすことができました。一方で、Figmaでは成立していても実装時に再現が難しい部分があり、調整に苦労しました。次回は、実装しやすさも意識してデザインを作成し、よりスムーズに開発へつなげたいです。

フロントエンド実装の振り返り 必須
HTML/CSS/JavaScript の実装について

フロントエンド実装では、企画やデザインで整理した内容をもとに、各ページの見た目や導線を実際のサイトとして形にできたことが良かったです。HTML/CSS/JavaScriptを使って、シーン別の商品一覧や商品詳細、ガイドページなどを実装し、サイト全体の統一感も意識して進めることができました。一方で、細かなレイアウト調整やレスポンシブ対応には想像以上に時間がかかりました。次回は、コンポーネント単位で整理しながら実装を進め、修正しやすい構成を意識したいです。

バックエンド実装の振り返り 必須
Java/Spring Boot の実装について

バックエンド実装では、JavaとSpring Bootを用いて商品管理機能や商品一覧・詳細表示機能を設計し、データベースと連携したWebアプリケーションとして形にできたことが良かったです。フロント側で見えていた画面の裏側に、どのようなデータ構造や処理が必要かを考えながら進めることで、システム全体の理解も深まりました。一方で、データベース設計や画面との連携、バリデーションの整理には苦労しました。次回は、実装前に処理の流れをより細かく整理し、安定して開発を進められるようにしたいです。

全体を通しての学び 必須
32日間のプロジェクト全体を通して学んだこと

32日間のプロジェクトを通して、企画、設計、デザイン、フロントエンド、バックエンドまで一連の流れを自分で経験できたことが大きな学びになりました。特に、見た目を作るだけでなく、ユーザーにとって使いやすい導線や、データの流れを意識して実装することの大切さを実感しました。また、事前の設計やルール整理が、その後の実装効率や修正のしやすさに大きく影響することも学びました。次回は、各工程のつながりをさらに意識しながら、より完成度の高い制作につなげたいです。

2 技術記事

技術記事タイトル 必須

HTML/CSS/JavaScriptでシーン別の商品一覧と商品詳細ページを設計・実装した工夫

記事本文 必須
上記の構成に沿って記事本文を記載。コードブロックを含めてください。

Bloom Letterのフロントエンド実装では、花の種類ではなく「シーンから選べる」体験をどう画面に落とし込むかを意識しました。特に商品一覧ページと商品詳細ページでは、見た目の美しさだけでなく、初めて利用する人でも迷いにくい導線設計を重視しました。

商品一覧ページでは、カードを並べるだけでなく、カテゴリやシーンから絞り込みやすい構成を意識しました。また、商品詳細ページでは価格、説明、おすすめシーン、配送目安などを整理し、購入前に必要な情報が自然に目に入るように調整しました。CSSでは余白や文字サイズ、カード幅のバランスを細かく見直しながら、サイト全体の統一感を保てるようにしています。

JavaScriptでは、表示切り替えやページネーションなど、一覧ページを使いやすくするための処理を実装しました。単に機能を追加するだけでなく、HTML構造とCSS設計が崩れないように、役割を分けながら記述することを意識しました。

.guide-support-grid {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 20px;
}

.detail-summary__price span {
  color: var(--site-text-sub);
  font-size: 0.75em;
  font-weight: 500;
}

このように、レイアウト比率や補足情報の見せ方を調整することで、ユーザーが情報を読み取りやすい画面に近づけました。今回の制作を通して、フロントエンドでは機能の実装だけでなく、細かな見た目の調整が使いやすさに大きく影響することを実感しました。今後は、再利用しやすいコンポーネント設計や保守しやすいCSS構成もさらに意識していきたいです。
