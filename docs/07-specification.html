1 システム概要

システムの目的 必須

Bloom Letter は、花の知識が少ないユーザーでも「贈るシーン」から商品を選べるフラワーギフトサイトである。バックエンドでは公開サイトに表示する商品情報の取得と、管理画面からの商品登録・編集・削除・公開制御を担う。

本仕様では公開サイトと管理画面の双方で必要になるデータ取得、入力チェック、認証、永続化処理を定義し、フロントエンドとバックエンドの責務境界を明確にする。

システム構成図 必須
Public Browser
  Home / Products / Product Detail / Contact
             |
             v
Spring Boot + Thymeleaf
  PublicController / AdminController / Service / Mapper
             |
     +-------+--------+
     |                |
     v                v
  MySQL/H2        Image Path

Admin Browser
  Login / Dashboard / Product Management
実装前提 必須

公開サイトは Spring Boot + Thymeleaf によるサーバーサイドレンダリングを前提とする。商品は 1 件につき 1 シーン所属、画像は 1 商品 1 メイン画像パスで管理する。

カート、決済、購入完了処理は本フェーズの実装対象外とし、商品詳細の購入導線は外部遷移またはダミー導線として扱う。

エラー応答方針 必須

画面系リクエストでは入力エラー時に同画面を再表示し、BindingResult による項目単位のメッセージを出す。存在しない商品や非公開商品の参照時は 404 を返す。

認証失敗時は `/admin/login?error` へリダイレクトし、CSRF 不正やサーバー例外は共通エラーページへ遷移する。

5 連携ポイント

フロントエンドとバックエンドの受け渡し項目 必須
画面フロントエンドで必要な値バックエンド提供方法備考
トップページ注目商品名、価格、画像、URLModel属性 `featuredProducts`4件固定表示
商品一覧商品カード配列、現在条件、ページ情報Model属性 `products`, `paging`検索条件は再表示時に保持
商品詳細商品名、価格、説明、画像、シーン名Model属性 `product`, `scene`非公開商品は公開サイトで非表示
Contact入力値、エラーメッセージ、完了状態Formオブジェクト + BindingResult送信失敗時は入力保持
商品登録/編集初期値、選択肢、エラーModel属性 `productForm`, `scenes`登録と編集で同一フォーム利用
連携ルール 必須

価格は整数で保持し、画面側で `¥5,800` 形式へ整形する。シーン識別は公開URLでは `sceneSlug`、管理画面では `sceneId` を使い分ける。

画像は `imagePath` を共通キーとして公開サイトと管理画面で参照する。公開可否は `isPublished`、トップ掲載可否は `isFeatured` で制御する。

6 非機能要件

セキュリティ要件 必須

管理画面は Spring Security により認証必須とし、パスワードは BCrypt で保存する。POST 系フォームには CSRF 対策を適用する。

性能・運用要件 必須

商品一覧は 1 ページ 12 件を基本とし、公開一覧の応答は 1 秒以内を目標とする。本番DBは MySQL、ローカルは H2 を想定する。

対応環境 必須

公開サイトは PC / タブレット / スマートフォン対応、管理画面は PC を主対象とする。対応ブラウザは Chrome、Edge、Safari の最新版を対象とする。
