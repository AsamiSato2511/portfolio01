1 システム概要

システムの目的 必須

BloomLetterの管理システムは、運営スタッフが商品情報や注文内容、問い合わせ状況を一元管理できるシステムです。更新や確認作業を効率化し、正確でスムーズな運営を支えることで、利用者へ安心感のある購買体験を提供します。

本仕様書では、公開サイトと管理画面で必要になる機能、画面遷移、入力チェック、APIの役割を整理し、BloomLetterの実装対象を明確にする。

     +-------+--------+

     v                v
   MySQL/H2       Image Path

Admin Browser
  Login / Dashboard / Product Management
実装前提 必須

公開サイトは Spring Boot + Thymeleaf によるサーバーサイドレンダリングを前提とする。商品は1件につき1シーンに所属し、商品画像はメイン画像パスを基準に表示する。

本フェーズでは商品閲覧、問い合わせ、管理画面での商品管理を対象とし、決済処理や配送管理、会員登録機能はスコープ外とする。

エラー応答方針 必須

画面系リクエストでは入力エラー時に同画面を再表示し、BindingResult による項目単位のメッセージを表示する。存在しない商品や非公開商品の参照時は404画面へ遷移する。

認証失敗時は /admin/login?error へリダイレクトし、CSRF不正やサーバー例外は共通エラーページで案内する。

5 連携ポイント

フロントエンドとバックエンドの受け渡し項目 必須
画面フロントで必要な値バックエンドからの提供形式備考
トップページ注目商品名、価格、画像、URLModel属性 featuredProducts4件固定表示
商品一覧商品カード配列、現在条件、ページ情報Model属性 products, paging絞り込み条件は再表示時に保持
商品詳細商品名、価格、説明、画像、シーン名Model属性 product, scene非公開商品は表示しない
Contact入力値、エラーメッセージ、完了状態Formオブジェクト + BindingResult送信失敗時は入力値を保持
商品登録/編集初期値、シーン選択肢、エラーModel属性 productForm, scenes登録と編集で同一フォームを利用
連携ルール 必須

価格は整数で保持し、画面側で ¥7,800 のように整形する。シーン識別子は公開URLでは sceneSlug、管理画面では sceneId を使用する。

画像は imagePath を共通キーとして公開サイトと管理画面で参照する。公開可否は isPublished、トップ表示可否は isFeatured で制御する。

6 非機能要件

セキュリティ要件 必須

管理画面は Spring Security による認証必須とし、パスワードは BCrypt でハッシュ化して保存する。POST系フォームにはCSRF対策を適用する。

性能・運用要件 必須

商品一覧は1ページ12件を基本とし、公開一覧の表示は1秒以内を目標とする。本番DBはMySQL、ローカル開発環境はH2を利用する。

対応環境 必須

公開サイトはPC、タブレット、スマートフォンに対応し、管理画面はPC利用を主対象とする。対応ブラウザは Chrome、Edge、Safari の最新版を対象とする。
